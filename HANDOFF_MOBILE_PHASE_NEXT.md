# SportsMate 모바일 다음 작업 인수인계

작성일: 2026-07-03

## 오늘 반영한 작업

- 모바일 채팅방 입력창 좌측에 `+` 버튼을 추가했습니다.
- `+` 메뉴에 사진 전송, 투표 생성, 위치 공유 버튼 껍데기를 추가했습니다.
- 투표 생성 버튼은 현재 사용자가 방장 또는 부방장 계열 권한일 때만 보이도록 조건을 넣었습니다.
  - 허용 역할: `host`, `cohost`, `subhost`, `assistant`
- 채팅방 응답에서 현재 사용자의 모임 참여 역할을 판단할 수 있도록 채팅방 `to_dict`에 `current_user_id`를 전달하는 흐름을 추가했습니다.
- 채팅 메시지의 상대방 프로필 이미지를 누르면 사용자 간략 정보 바텀시트가 뜨는 껍데기를 추가했습니다.
  - 표시 정보: 프로필 이미지, 닉네임, 지역
  - 준비 버튼: `1:1 쪽지`, `차단`
- 채팅방 더미 공지 접기/펼치기 UI를 유지했습니다.
- 채팅 입력창 아이콘 배치가 깨지던 문제를 3열 그리드로 고정했습니다.
- Supabase metadata 덮어쓰기 완화 목적으로 바꿨던 프로필 이미지 우선순위 변경은 원래 방향으로 되돌렸습니다.

## 다음 작업 Phase

### Phase 1. 채팅 부가 기능 연결

- 사진 전송 버튼 실제 기능 연결
  - Supabase Storage 또는 Flask 업로드 API 중 하나로 결정해야 합니다.
  - `chat_messages.message_type`에 `image`를 저장하는 방식 검토가 필요합니다.
- 위치 공유 버튼 실제 기능 연결
  - 좌표, 장소명, 주소를 담는 메시지 타입 설계가 필요합니다.
  - 지도 API와 연결할 경우 VWorld 또는 기존 지도 검색 흐름과 맞춰야 합니다.
- 투표 생성 버튼 실제 기능 연결
  - 프론트 버튼은 권한별 노출 조건만 적용된 상태입니다.
  - 백엔드에서도 방장/부방장 권한 검증이 필요합니다.
  - 투표 생성 후 채팅방에 투표 카드 메시지를 보여주는 구조를 검토해야 합니다.

### Phase 2. 채팅 사용자 액션

- 사용자 프로필 바텀시트의 `1:1 쪽지` 기능 연결
  - 1:1 쪽지를 별도 DM 테이블로 갈지, 기존 채팅방 구조를 재사용할지 결정해야 합니다.
- 사용자 차단 기능 연결
  - 차단 테이블 또는 관계 테이블이 있는지 확인해야 합니다.
  - 차단 시 채팅 메시지 숨김, 모임 신청 제한 등 정책 결정이 필요합니다.

### Phase 3. 공지 데이터 연동

- 현재 공지는 더미 데이터입니다.
- `notices` 테이블 또는 모임 공지 API를 확인한 뒤 최신 공지 1개를 채팅방 상단에 표시해야 합니다.
- 공지가 없으면 공지 영역은 숨기는 것이 좋습니다.
- 공지 목록은 추후 바텀시트 또는 모달로 확장할 수 있습니다.

### Phase 4. 모바일 실기기 검증

- 모바일 Safari/Chrome에서 입력창 자동 확대 여부 확인
- 키보드가 올라왔을 때 채팅 입력창과 하단 네비가 겹치지 않는지 확인
- 새 메시지 전송/수신 시 스크롤이 하단에 유지되는지 확인
- `+` 메뉴가 작은 화면에서도 잘리지 않는지 확인

### Phase 5. 배포 성능 최적화

- 현재 Vite 빌드에서 JS 번들이 800KB 이상으로 경고가 발생합니다.
- 라우트 단위 `React.lazy` 코드 스플리팅을 적용하면 Vercel 초기 로딩을 줄일 수 있습니다.
- 관리자/PC/모바일 주요 화면을 우선 분리하는 방향이 좋습니다.

## 주의 사항

- PC 디자인과 레이아웃에는 간섭하지 않아야 합니다.
- 모바일 작업은 가능한 한 `frontend/src/components/**/mobile` 및 `frontend/src/styles/17-mobile-stitch.css` 범위에서 처리합니다.
- 백엔드 공통 API를 수정할 때는 PC 담당자 작업과 충돌 가능성을 먼저 확인해야 합니다.
- 실제 권한 검증은 프론트 조건만으로 끝내면 안 되고 백엔드에서도 반드시 검증해야 합니다.
